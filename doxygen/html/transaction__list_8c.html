gboolean transaction_list_remove_transaction ( gint  transaction_number  ) 

remove a transaction from the tree model this doesn't change any filter or sort parameters, so just need to colorize and set the balances after that

WARNING : the transaction (and its children) must not be freed when come here

NOTE : if the removed transaction is the selected transaction, the model won't have selected transaction anymore, so need to do selection_next before

Paramètres:
transaction_number the transaction to remove
Renvoie:
TRUE : transaction removed, FALSE : problem, nothing done
